# Demo 影片錄製腳本

> 建議分三段錄製後剪接，全片控制在 3～4 分鐘以內

---

## 準備工作（錄影前）

```bash
# 確保所有容器是最新版本
docker compose up --build -d

# 確認四個容器都在跑
docker compose ps
```

- 瀏覽器開 `http://localhost:8080`
- 清空任何舊任務
- 終端機視窗放在旁邊備用（Part 3 要用）

---

## Part 1：基礎功能（約 90 秒）

### ① 介面總覽（10 秒）
- 慢慢滑動整個頁面，讓鏡頭掃過：方案選擇列、Canvas、節點狀態卡、任務列表
- 三個節點卡都顯示 **idle**

### ② 提交第一個任務（20 秒）
- 點選「小超市」方案
- 調整人數到 **8**，步數改 **200**
- 點「提交任務」
- 鏡頭停在任務列表：看到 `queued → running`，節點卡上出現 **busy + job ID**

### ③ 節點監控（10 秒）
- 放大鏡頭對準節點卡
- 展示 CPU% 和 MEM% 數字在跳動

### ④ 看結果（20 秒）
- 任務變成 done → 點「查看 GIF」
- 讓 GIF 播幾秒
- 點「查看資訊」展開 stats 面板，滑一下購物記錄

### ⑤ 刪除排隊中的任務（30 秒）
- 快速連點提交 **3 個任務**（讓三個節點都 busy）
- 再提交第 4 個 → 第 4 個出現在佇列顯示 `queued`
- 點第 4 個任務的刪除按鈕 → 狀態變 `cancelled`
- 旁白：「只有排隊中的任務可以刪除，執行中的不行」

---

## Part 2：進階功能（約 60 秒）

### ⑥ Canvas 地圖編輯（20 秒）
- 選「小小超市」→ 畫面縮成 20×20
- 用滑鼠在空白處拖曳畫牆（右鍵清除）
- 加一個新貨架格、多加一個入口
- 展示 Canvas 和地圖真的改變了

### ⑦ 進階參數面板（20 秒）
- 點「進階 ▾」展開
- 把 **goal_reserve** 從 10 改成 **5**
- 把 **spawn_interval** 改成 **3**
- 提交任務，旁白說明這些參數的意義

### ⑧ 方案對比（20 秒）
- 切換到「大型賣場」方案
- 讓鏡頭看到 Canvas 自動從 20×20 擴展到 60×60
- 強調地圖差異

---

## Part 3：節點當機偵測與重分配（約 40 秒）

### ⑨ 填滿節點（10 秒）
- 提交 3 個「中型商場」任務
- 等三個節點都顯示 **busy**，停留幾秒讓鏡頭拍到

### ⑩ 模擬節點當機（30 秒）
- 切到終端機，打指令但先不按 Enter：
  ```bash
  docker stop node1
  ```
- 回到瀏覽器畫面
- 按 Enter 執行
- 鏡頭一直盯著網頁，約 **12 秒**後：
  - node1 節點卡變成 **offline**
  - 原本在 node1 的任務狀態從 `running` 變回 `queued`
- 再等幾秒，node2 或 node3 完成後自動接走那個任務
- 最後執行：
  ```bash
  docker start node1
  ```
- node1 回到 **idle**，展示自動恢復

| 段落 | 長度 | 重點字幕建議 |
|------|------|------------|
| Part 1 | ~90 秒 | 標出 `queued → running → done` 狀態轉換 |
| Part 2 | ~60 秒 | 圈出進階面板、Canvas 尺寸變化 |
| Part 3 | ~40 秒 | 紅框標出 node1 從 `busy → offline`，任務自動重排 |

## 錄影工具

- **Windows 內建**：`Win + G` Xbox Game Bar → 開始錄製
- **OBS Studio**：畫質較好，可加字幕疊層
- 建議解析度：1920×1080，錄製瀏覽器視窗即可
