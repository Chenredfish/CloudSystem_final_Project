Dispatcher

  • 找 idle 節點 + pending job
  • BEGIN IMMEDIATE 交易鎖，防止搶單
  • 背景 scanner 每 5s 掃描
  • 節點 offline → job 重排為 pending
  • 節點重啟後孤立 running job → 重排

Heartbeat

  • 節點每 5 秒回報一次
  • 帶 CPU% 與 MEM%
  • 超過 15 秒未回報 → offline
  • Web 節點卡顯示 → scanner 自動恢復

SQLite WAL Mode — 為什麼夠用？

  • Write-Ahead Log：寫入不鎖整個檔案
  • 多個讀取者可同時讀（無讀讀鎖）
  • Dispatcher 用 BEGIN IMMEDIATE 確保寫入原子性
  • 3 節點並發量小，WAL 性能完全足夠

核心算法
PBS — Priority-Based Search
固定優先序、逐層規劃的多代理人路徑搜索

MAPF 衝突種類

頂點衝突：兩人在同一時間佔同一格
✅ PBS 完全避免
交換衝突（對穿）：兩人互換相鄰格
✅ 本專案已修正（edge_reserved）

PBS 規劃流程

  1. 依 agent 索引排定固定優先序
  2. 高優先 agent 先執行 Time-Expanded A*
  3. 將其路徑加入 reserved 集合
  4. 低優先 agent 的 A* 必須繞開 reserved
  5. 每個模擬步驟重新規劃一次

算法特性

優點
  • 實作簡單，速度快
  • 頂點衝突完全消除
  • 適合教學展示
限制
  • 不保證全域最優解
  • 低優先 agent 可能顯著變慢
  • 固定優先序對 agent 0 有利
更優算法：CBS（最優解）或 WHCA*（滑動視窗重規劃）
本系統已規劃 WHCA* 升級路線

算法細節
Time-Expanded A* 與六項改進

Time-Expanded A*

搜索空間：(r, c, t) 三維
每個狀態為「在 t 時刻位於格子 (r,c)」
允許等待（原地停留）
max_t 限制搜索深度
  • 障礙：WALL + 非目標的 SHELF/CASHIER
  • reserved 集合：高優先 agent 的 (r,c,t)
  • Heuristic：Manhattan 距離
  • 回傳 None → agent 原地等待此步

關鍵參數

  • goal_reserve：抵達目標後保留格子步數
  • min_t_floor：A* 最短時間視窗
  • spawn_interval：每批進場間隔步數

本次實作的六項改進

1
BFS 可達性預檢：模擬前驗證所有貨架/收銀台從入口可達，不可達立即報錯
2
goalReserve 自動上限：clamp 至 max_steps÷4，防止收銀台永久死鎖
3
貪婪最近鄰清單排序：從入口出發，依距離重排購物順序，減少來回折返
4
對穿衝突修正：edge_reserved 記錄每個移動的反向邊，A* 展開時禁止互穿
5
動態收銀台選擇：購物完成時才選收銀台，分數 = 距離 + 排隊人數 × goal_reserve
6
入口擁塞重試：等待逾 spawn_interval×2 步後自動改走備用入口

Q & A
常見問題整理
Q1. PBS 為什麼會超時？
低優先 agent 必須繞開所有人的保留路徑，人數越多可用路徑越少。 goal_reserve 過高時，收銀台被鎖太久，後到者 A* 看不到空位，只能停在原地。
Q2. 節點掛掉工作會怎樣？
Heartbeat 超時後 background_scanner 將對應節點標為 offline， 該節點的 running job 重排為 pending，由其他節點接手。
Q3. SQLite 能支援多節點並發嗎？
WAL mode 允許多讀者同時讀，Dispatcher 用 BEGIN IMMEDIATE 確保同一 job 不會被兩個節點同時搶走。3 節點量級完全足夠。
Q4. 為什麼選 PBS 不是 CBS？
PBS 實作簡單、速度快，適合教學展示。 CBS 可保證最優解但複雜度顯著更高。本系統已規劃 WHCA* 作為升級路線。
Q5. WHCA* 和 PBS 的差別？
WHCA* 以滑動視窗（window）重規劃，所有 agent 在每個視窗內平等競爭， 解決 PBS 固定優先序對 agent 0 過度有利的問題，公平性更好。
Q6. 系統可以水平擴展嗎？
可以，新增 node4 只需在 docker-compose.yml 複製一個 node 服務並設定 SELF_ID 環境變數，web 的 Dispatcher 發現新節點 heartbeat 後會自動排程。
